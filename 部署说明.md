## 部署方法

### 方法一：使用部署脚本（推荐）
```bash
# 赋予执行权限
chmod +x 部署脚本.sh

# 执行部署
./部署脚本.sh
```

### 方法二：手动部署
```bash
# 1. 连接到主服务器
ssh root@106.14.237.27

# 2. 进入项目目录
cd /var/www/fu/

# 3. 创建备份
mkdir -p /var/backups/fu
cp index.html /var/backups/fu/backup_$(date +%Y%m%d_%H%M%S).html

# 4. 上传新文件
# 在本地终端执行：
scp 符箓生成器_源代码.html root@106.14.237.27:/var/www/fu/index.html

# 5. 设置权限
chmod 644 index.html
chmod -R 755 images/

# 6. 重启服务
systemctl restart nginx

# 7. 对备用服务器重复以上步骤
ssh root@172.24.19.113
cd /var/www/fu/
# ... 重复步骤3-6
```

## SSH连接问题解决方案

### 1. 检查SSH密钥
```bash
# 检查是否有SSH密钥
ls -la ~/.ssh/

# 如果没有，生成新的SSH密钥
ssh-keygen -t rsa -b 4096

# 将公钥复制到服务器
ssh-copy-id root@106.14.237.27
```

### 2. 使用密码认证
```bash
# 如果SSH密钥不可用，可以使用密码认证
ssh -o PreferredAuthentications=password root@106.14.237.27
```

### 3. 检查网络连接
```bash
# 测试网络连接
ping 106.14.237.27

# 测试SSH端口
telnet 106.14.237.27 22
```

## 常见问题

### 1. Permission denied
- 检查SSH密钥是否正确配置
- 确认用户名和密码是否正确
- 检查服务器是否允许SSH连接

### 2. Connection refused
- 检查服务器是否在线
- 检查防火墙设置
- 检查SSH服务是否运行

### 3. File not found
- 检查文件路径是否正确
- 确认文件是否存在
- 检查权限设置

### 4. Nginx错误
- 检查Nginx配置
- 查看错误日志：`tail -f /var/log/nginx/error.log`
- 重启Nginx：`systemctl restart nginx`
