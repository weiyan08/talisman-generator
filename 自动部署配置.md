## 🚀 自动部署方案

### 方案一：GitHub Actions（推荐）
创建 `.github/workflows/deploy.yml` 文件：

```yaml
name: 自动部署符箓生成器

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
      
    - name: Deploy to production
      uses: appleboy/ssh-action@v0.1.0
      with:
        host: ${{ secrets.SSH_HOST }}
        username: ${{ secrets.SSH_USERNAME }}
        key: ${{ secrets.SSH_PRIVATE_KEY }}
        script: |
          cd /var/www/fu
          cp index.html index.html.backup.$(date +%Y%m%d_%H%M%S)
          cp ../talisman-generator/index.html .
          chmod 644 index.html
          systemctl restart nginx
```

### 方案二：Webhook + 自动脚本
创建 `webhook-deploy.sh`：

```bash
#!/bin/bash
# webhook-deploy.sh - Webhook触发自动部署

# 服务器配置
SERVERS=("106.14.237.27" "172.24.19.113")
PROJECT_PATH="/var/www/fu"

# GitHub Webhook secret
WEBHOOK_SECRET="your_webhook_secret"

# 验证Webhook请求
signature=$(echo "$1" | openssl dgst -sha1 -hmac "$WEBHOOK_SECRET" | sed 's/^.* //')
if [ "$signature" != "$HTTP_X_HUB_SIGNATURE_1" ]; then
    echo "Webhook signature verification failed"
    exit 1
fi

# 部署到每个服务器
for server in "${SERVERS[@]}"; do
    echo "部署到服务器: $server"
    ssh root@$server "cd $PROJECT_PATH && cp index.html index.html.backup.$(date +%Y%m%d_%H%M%S) && git pull origin main && cp talisman-generator/index.html . && chmod 644 index.html && systemctl restart nginx"
done

echo "部署完成"
```

### 方案三：定时自动检查（简单）
创建 `auto-deploy.sh`：

```bash
#!/bin/bash
# auto-deploy.sh - 定时自动部署检查

# 配置
REPO_URL="https://github.com/weiyan08/talisman-generator.git"
LOCAL_REPO="/tmp/talisman-generator"
PROJECT_PATH="/var/www/fu"
SERVERS=("106.14.237.27" "172.24.19.113")

# 检查是否有更新
check_updates() {
    cd $LOCAL_REPO
    git fetch origin
    
    if [ $(git rev-list HEAD...origin/main --count) -gt 0 ]; then
        echo "发现更新，开始部署..."
        deploy
    else
        echo "没有更新"
    fi
}

# 部署函数
deploy() {
    # 拉取最新代码
    git pull origin main
    
    # 部署到服务器
    for server in "${SERVERS[@]}"; do
        echo "部署到服务器: $server"
        scp $LOCAL_REPO/index.html root@$server:$PROJECT_PATH/
        ssh root@$server "chmod 644 $PROJECT_PATH/index.html && systemctl restart nginx"
    done
}

# 主循环
while true; do
    check_updates
    sleep 3600  # 每小时检查一次
done
```

## 🔧 配置步骤

### 1. GitHub Actions配置
```bash
# 创建GitHub Actions工作流目录
mkdir -p .github/workflows

# 创建部署工作流文件
cat > .github/workflows/deploy.yml << 'EOF'
name: 自动部署符箓生成器

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
      
    - name: Deploy to production
      uses: appleboy/ssh-action@v0.1.0
      with:
        host: ${{ secrets.SSH_HOST }}
        username: ${{ secrets.SSH_USERNAME }}
        key: ${{ secrets.SSH_PRIVATE_KEY }}
        script: |
          cd /var/www/fu
          cp index.html index.html.backup.$(date +%Y%m%d_%H%M%S)
          cp talisman-generator/index.html .
          chmod 644 index.html
          systemctl restart nginx
EOF

# 提交配置
git add .github/workflows/deploy.yml
git commit -m "添加GitHub Actions自动部署"
git push origin main
```

### 2. 配置GitHub Secrets
在GitHub仓库设置中添加以下Secrets：
- `SSH_HOST`: 服务器IP (106.14.237.27)
- `SSH_USERNAME`: root
- `SSH_PRIVATE_KEY`: SSH私钥内容

### 3. 启动自动部署服务

## 🎯 推荐方案

**推荐使用GitHub Actions**，原因：
1. **官方支持**：GitHub官方维护

3. **触发机制灵活**：push、pr、schedule等
4. **日志清晰**：完整的部署日志
5. **安全可靠**：Secrets管理
